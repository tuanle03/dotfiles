# cloudflared

Personal Cloudflare Tunnel configs. Two named tunnels expose four local dev services to public HTTPS hostnames via Cloudflare's edge — no port-forwarding, no static IP.

## What runs where

| Tunnel  | Hostname              | Local target          | Use                          |
| ------- | --------------------- | --------------------- | ---------------------------- |
| `cnmeo` | `lixicrm.cnmeo.site`  | `http://localhost:3000` | Chatwoot (Rails)             |
| `cnmeo` | `lixfollow.cnmeo.site`| `http://localhost:4000` | LixFollow API (FastAPI)      |
| `cnmoew`| `survey.cnmoew.site`  | `http://localhost:3000` | Survey app                   |
| `cnmoew`| `cnmoew.site`         | `http://localhost:2106` | Apex app                     |

Both `cnmeo` + `cnmoew` zones live in the same Cloudflare account, both DNS records are proxied (orange cloud), CNAME → `<tunnel-uuid>.cfargotunnel.com`.

## Files

| Path                              | Purpose                                    |
| --------------------------------- | ------------------------------------------ |
| `config.yml`                      | `cnmeo` tunnel — the **default** that `cloudflared tunnel run cnmeo` reads from `~/.cloudflared/config.yml` |
| `cnmoew.yml`                      | `cnmoew` tunnel — passed explicitly with `--config` |

Both reference `credentials-file: ~/.cloudflared/<UUID>.json` — the per-tunnel secret JSON that lives only on disk (gitignored), generated by `cloudflared tunnel create`.

`~/.cloudflared/cert.pem` is the **origin certificate** — issued by `cloudflared tunnel login` and authorizes managing tunnels + DNS for the zones selected during login. Also not committed.

## Daily use

```bash
cloudflared tunnel run cnmeo                              # uses ~/.cloudflared/config.yml
cloudflared tunnel --config ~/.cloudflared/cnmoew.yml run # uses cnmoew.yml's tunnel field
```

The tmuxinator sessions `lixfollow` and `lixicrm` start these automatically — see `tmuxinator/lixfollow.yml` and `tmuxinator/lixicrm.yml`.

## Bootstrap on a fresh machine

The configs in this repo are **machine-specific** — the `credentials-file` paths embed UUIDs that only exist on the Mac that originally ran `cloudflared tunnel create`. On a new machine, you must mint new tunnels and edit the configs to point at the new UUIDs.

```bash
# 1. Install
brew install cloudflared

# 2. Authorize both zones in one go (multi-select in the browser)
cloudflared tunnel login        # select cnmeo.site AND cnmoew.site
                                # cert lands at ~/.cloudflared/cert.pem

# 3. Create the two tunnels
cloudflared tunnel create cnmeo
cloudflared tunnel create cnmoew
# each prints a UUID + writes ~/.cloudflared/<UUID>.json (the secret)

# 4. Symlink configs from this repo
ln -sf ~/projects/dotfiles/cloudflared/config.yml  ~/.cloudflared/config.yml
ln -sf ~/projects/dotfiles/cloudflared/cnmoew.yml  ~/.cloudflared/cnmoew.yml

# 5. Edit credentials-file paths to the NEW UUIDs
nvim ~/.cloudflared/config.yml ~/.cloudflared/cnmoew.yml

# 6. Route the hostnames at the new tunnels — USE UUID (see Gotchas)
cloudflared tunnel route dns --overwrite-dns <cnmeo-uuid>  lixicrm.cnmeo.site
cloudflared tunnel route dns --overwrite-dns <cnmeo-uuid>  lixfollow.cnmeo.site
cloudflared tunnel route dns --overwrite-dns <cnmoew-uuid> survey.cnmoew.site
cloudflared tunnel route dns --overwrite-dns <cnmoew-uuid> cnmoew.site

# 7. Smoke test
cloudflared tunnel run cnmeo &
curl https://lixfollow.cnmeo.site/health
```

To run as a launchd daemon (auto-start on login):

```bash
sudo cloudflared --config ~/.cloudflared/config.yml service install
```

(The service installer only handles the default `config.yml` — for `cnmoew` add a second LaunchAgent plist manually, or run it from tmuxinator as today.)

## Gotchas

**Fuzzy-match bug — always use UUID for `route dns`.** On v2026.5.0, `cloudflared tunnel route dns cnmoew <host>` silently resolves the name `cnmoew` to the lexically close tunnel `cnmeo`. Result: the CNAME points to the wrong tunnel's `.cfargotunnel.com` target. Workaround: pass the UUID instead of the name.

```bash
# WRONG — created a CNAME pointing at cnmeo
cloudflared tunnel route dns cnmoew survey.cnmoew.site

# RIGHT — UUID is unambiguous
cloudflared tunnel route dns 3e6280ed-... survey.cnmoew.site
```

`run`, `info`, `delete` may have the same fuzzy behavior. When in doubt, use the UUID. Verify after with:

```bash
dig +short <hostname> A   # proxied → returns Cloudflare edge IPs (not the CNAME)
```

…or hit the Cloudflare API directly to read the CNAME `content`.

**Cert is per-zone, not per-account.** `cloudflared tunnel login` writes a single cert.pem authorized for whichever zones you ticked in the browser. If the cert only covers `cnmeo.site` and you try `route dns ... survey.cnmoew.site`, cloudflared treats `cnmoew.site` as a **label inside** `cnmeo.site` and creates the bogus record `survey.cnmoew.site.cnmeo.site`. To fix, re-login (move the old cert.pem aside first — cloudflared refuses to overwrite) and tick both zones.

**Overwriting existing DNS records.** `route dns` errs out if a non-tunnel CNAME/A already exists at that hostname. Add `--overwrite-dns` to force.

**Apex record (`cnmoew.site`).** Cloudflare's CNAME flattening lets a tunnel CNAME live at the zone apex transparently. No special config needed beyond the normal `route dns`.

**Same local port across tunnels is fine but suspicious.** `lixicrm.cnmeo.site` and `survey.cnmoew.site` both proxy to `localhost:3000`. Only one app can bind that port at a time — so today both hostnames serve whatever is on :3000 (typically Chatwoot). Adjust if you actually want them to be different apps.

## Operations cheatsheet

```bash
cloudflared tunnel list                          # show all tunnels
cloudflared tunnel info <name-or-uuid>           # show active connections + edge POPs
cloudflared tunnel ingress validate              # validate ~/.cloudflared/config.yml
cloudflared tunnel --config X ingress validate   # validate a specific config

# DNS via Cloudflare API (cert.pem's apiToken can manage DNS in authorized zones)
python3 -c '
import base64, json
b64 = "".join(l for l in open("/Users/tuanle/.cloudflared/cert.pem").read().splitlines()
              if "ARGO TUNNEL TOKEN" not in l)
print(json.loads(base64.b64decode(b64)))'
```

`cloudflared` has no `route dns delete` subcommand — to remove a CNAME, use the Cloudflare dashboard or the API.
