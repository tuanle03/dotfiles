## Overview

Personal dotfiles for Neovim, Vim, Tmux, and Zsh. All configs live here and are linked into the home directory via symlinks. On a new machine: clone the repo then run `make install`.

## Bootstrap a New Machine

```bash
git clone git@github.com:tuanle03/dotfiles.git ~/dotfiles
cd ~/dotfiles
make install
```

`make install` runs `scripts/install.sh`, which:
- Creates symlinks for all configs (backs up any existing files as `.bak`)
- Copies `zsh/.zshrc.secrets.example` → `~/.zshrc.secrets` if not present
- Installs `packer.nvim` if not already installed

After install:
1. Add API keys to `~/.zshrc.secrets` (gitignored)
2. Open nvim and run `:PackerSync`
3. Run `tmux source-file ~/.tmux.conf`

## Syncing Configs Back to the Repo

```bash
make sync        # or: bash scripts/sync.sh
```

Since all configs are symlinked, edits made in nvim/tmux are already reflected in the repo. `sync.sh` is primarily useful for pulling in tmuxinator project files (`.yml`) created or edited via tmuxinator directly, and for non-symlinked setups.

After syncing: `git diff` to review, then commit.

## Repository Structure

```
dotfiles/
├── nvim/           # → ~/.config/nvim        (Neovim config)
├── vim/            # → ~/.vimrc etc           (legacy Vim config)
├── tmux/           # → ~/.tmux.conf etc       (Tmux config)
├── tmuxinator/     # → ~/.config/tmuxinator   (session definitions)
├── zsh/
│   ├── .zshrc                  # → ~/.zshrc (no secrets)
│   └── .zshrc.secrets.example  # template → copy to ~/.zshrc.secrets
├── scripts/
│   ├── install.sh  # bootstrap: symlink everything
│   └── sync.sh     # pull live configs back into repo
├── Makefile        # make install / make sync
└── .gitignore      # excludes *.secrets, packer_compiled.lua, .DS_Store
```

## Neovim Config Load Order

Understanding this is key to knowing where to make changes:

1. **`nvim/init.lua`** — Core options + plugin setup (gitsigns, CopilotChat, nvim-tree, lualine, moiday). Dynamically loads the next two files at `~/.config/nvim/`.
2. **`nvim/plugin/bundles.lua`** — Packer.nvim plugin declarations. Add/remove plugins here.
3. **`nvim/local.vim`** — All keymaps, FZF/ripgrep commands, plugin variables, Rails-specific bindings.
4. **`nvim/plugin/packer_compiled.lua`** — Auto-generated by Packer; gitignored, do not edit.

## Key Conventions

- **Leader key:** `<Space>`
- **Tmux prefix:** `C-Space` (replaces default `C-b`)
- **Plugin manager:** Packer.nvim (Neovim)
- **Search:** ripgrep (`rg`) → `fd` → `ag` fallback chain
- **Colorscheme:** `catppuccin-macchiato` (07:00–18:00) / `catppuccin-mocha` (evening), auto-set on `VimEnter`
- **File tree:** nvim-tree aliased to NERDTree commands for compatibility

## Secrets

API keys (OpenAI, Anthropic, Gemini, etc.) are **never committed**. They live in `~/.zshrc.secrets`, sourced at the end of `.zshrc`. Use `zsh/.zshrc.secrets.example` as the template.

## Requirements

- Neovim 0.8+, Tmux 3.0+, Zsh 5.0+
- [Nerd Fonts](https://www.nerdfonts.com/) for icons in lualine/nvim-tree
- `rg` (ripgrep) for FZF integration
- `tmuxinator` gem for session management
